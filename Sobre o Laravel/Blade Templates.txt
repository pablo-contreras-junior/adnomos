Os Modelos Blade nos permitem disponibilizar páginas web com o Laravel de forma facil e organizada:

    Algumas tags importantes:
	@yield()
	@extends()
	@section() e @endsection

   Para que elas servem?

	@yield('nome'):
	   |
	   V
	Quando criamos uma estrutura HTML contendo a base de todas as outras páginas, a tag @yields serve 
	para específicar as partes do modelo que irão variar em cada págína. Exemplo: ao criar uma estrut-
	rutura HTML que contenha a tag <main></main>, podemos especificar o conteudo que teremos na tag
	<main></main> de cada template usando @yield('nome_do_conteudo'), que posteriormente será substit-
	uido pelo vaalor da tag @section().



	@section('nome_do_conteudo_substituivel') e @endsection — se necessário:
	   |
	   V
	Essa tag nos permite substituir determinado conteúdo de uma tag @yield pelo que for necessário.
	Tudo o que colocarmos dentro de @section e @endsection será será unico dentro do template que est-
	amos criando. O parâmetro de @section() serve para especificar a tag @yield que vamos modificar.
	Se preferirmos, podemos especificar o contdudo que queremos adicionar no parâmetro de @section e 
	nesse caso, não precisaremos da tag @endsection.



	@extends('nome_da_estrutura_principal'):
	   |
	   V
	Nos permite herdar a estrutura e as tags @yield do modelo principal. Essa tag busca todas as tags
	HTML (e tags de template) do layout principal e traz para a página que queremos montar.
